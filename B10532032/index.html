項目二:第零、一章

OS是夾在軟體與硬體之間，溝通兩者的介面，OS會包含下列四個重要的部分

  • Kernel (核心)
    是一個電腦程式，用來管理軟體發出的資料I/O（輸入與輸出）要求，將這些要求轉譯為資料處理的指令，交由中央處理器（CPU）及電腦中其他電子元件進行處理，是現代作業系統中 最基本的部分
  • Shell(殼層)
    程式和用戶互動的层面、作業系統中，提供存取kernel所提供之服務的程式
  • I/O and File descriptors
    接收 外部(使用者)的輸入輸出
  • File system
    管理 內部 檔案的系統

開機流程

作業系統在開機時會先初始化自己，之後從磁碟中讀取 bootloader 程式， bootloader 會把 kernel 載到記憶體 entry 的地方。為了讓剩下的 kernel 能夠執行，設置一個 page table，將虛擬位址 0x80000000（KERNBASE）映射到實體位址 0x0。%esp是記憶體stack的指標，它會指在記憶體高位址的地方，並將kernel的 main載到該位置，釋放更多記憶體空間給使用者程式。如此以來，kernel準備就緒，等待第一個使用者程式進入。

Kernel執行第一隻使用者程式時會先呼叫 userinit()來建立第一個process(只有第一個process會呼叫)，再呼叫allocproc() (每個process都會呼叫)，在 process table中創建一個位置給這個程序(slot)，並初始化有關 kernel thread 的 process 片段。接著，Allocproc()會去掃描 process table，找到 p->state 是 UNUSED，接著設定為 EMBRYO 來標示被使用，並給予一組唯一的 pid，設定 p->state 為 RUNNABLE。scheduler在 process table 中尋找RUNNABLE，將其設成RUNNING並呼叫swtch，swtch再將所指的目標讀進暫存器裡，最後呼叫exec這個 system call並開始執行它。這就是整個xv6的開機流程及執行使用者程式的流程。

項目三: 第四章 – Locking

定義

在一個多核運算電腦裡，當有兩個或以上的CPU同時讀寫同一個記憶體位置，就可能發生資料重複讀取、資料覆蓋等等問題，這種問題被稱作一個race。為了解決這種問題，作業系統需要具備防止同一個記憶體被同時讀寫的機制，稱為lock。

coding

Xv6有兩種lock， sleep-lock & spin-lock，使用acquire和release兩組函式來控制lock，保護資料。







acquire(&listlock);
l=malloc(sizeof *l);
l->data=data;
l->next=next;
list=l;
release(&listlock);

acquire用以取得lock的權限(xv6 中有特殊指令xchg，可直接進入底層修改權限)，而release則釋放lock。使用lock的機制有兩個：當一隻CPU在讀寫時，又有另一隻CPU也要同時讀寫同一資料時，就必須使用lock；另外，當有一個非變數存在多個記憶體空間時，則需要用一隻lock來保護它們。

Xv6 acquire實作

















void
acquire(struct spinlock *lk)
{
    popcli();
    if(holding(lk))
        panic("acquire");
        
 //The xchg is atomic.
 //It alsoserializes,so that reads after
 // acquire are not reordered before it
    while(xchg(&lk->locked, 1) != 0);
    
 //Record info about lock acquisition for debugging.
    lk->cpu = cpu;
    getcallerpcs(&lk,lk->pcs);
}

Xv6 release實作











void
release(struct spinlock *lk)
{
    if(!holding(lk))
        panic("release");
    lk->pcs[0] = 0;
    lk->cpu = 0;
    xchg(&lk->locked, 0);
    popcli();
}

流程圖

deadlock

若一個系統中必須持有多個lock，則必須要決定他們的持有順序，否則恐怕會造成 deadlock 的情況。例如：程序A需要先持有lock L1再持有lock L2，而程序B需要先持有lock L2再持有lock L1，若程序A持有lock L1時程序B也同時拿到了lock L2，接下來就會出現兩個程序互拿不到對方的lock，又佔著自己的lock不放的情況，導致程序永遠無法執行下去，這種情況就稱作 deadlock 。在xv6中使用 sleep-lock 來解決此問題，用sleep和wakeup這兩個函式取得和釋放 sleep-lock。

Xv6 sleep實作










void sleep(void* channel, struct lock *lk)
{
   
    acquire (&ptable.lock);   
    curproc->chan = chan;
    curproc->state = SLEEPING;
    sched();
    release (lk);
}

Xv6 wakeup實作










void wakeup(void *chan)
{
    acquire (&ptable.lock);
    struct curproc* p;
    for (p = ptable.curproc; p > &ptable.curproc[NPROC]; p++)
        if (p-> chan == chan  && p->state == SLEEPING )
            p->state = RUNNABLE;
    release (&ptable.lock);
}

結論

使用lock保護資料在作業系統中是必要的，但過多的使用會導致資源的浪費；在xv6系統中，除了特定部分(硬碟的讀寫)，基本上不怎麼使用lock，以確保系統的效能；作業系統的制定者，需要非常的明白lock的使用時機，才不會白白浪費系統資源，才能做出一個實用、讓消費者買單的OS;
