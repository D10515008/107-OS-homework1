項目二

作業系統(Operation System)是管理電腦硬體的軟體程式，也是硬體與軟體間的溝通橋樑，提供使用者簡單操作介面來控制並管理硬體。有些作業系統講求簡單操作，有些則講求效率性。設計一個好的使用者介面(Interface)需要仰賴一些機制，透過這些機制才能做出強大的功能性。

作業系統的運作基本元件是核心(Kernel)，當電腦程序需要調用核心的服務時，作業系統會透過System Call 來調用核心，才能讓電腦使用到核心的功能。而作業系統最主要功能有幾個

  1. 管理程式的執行
  2. 有效的分配系統資源
  3. 控制I/O裝置
  4. 檔案管理及網路管理
  5. 提供使用者簡單操作介面

項目三 Chapter 2 Page table

Page table基本介紹

Page table是一種在作業系統中控制記憶體的機制。Page table能夠讓虛擬位址對應到實體記憶體位址上，當一個程序要求要使用到記憶體，作業系統負責分配虛擬位址到實體位址上，所有的對映項目就稱為Page table。且還提供多工的功能，讓不同程序能同時在記憶體執行，並保護記憶體中程序間不互相影響的保護機制。

Paging hardware

  • x86裡，每個page table包含2^20個Page table entries()，每個PTE裡有20bit的page number(PPN)及12bit可表示一些狀態的flag組成。如下圖所示。

  • 實體記憶體中，則是以two-level tree方式來儲存。Paging hardware會使用虛擬位址前10bit來選擇page directory entry，接著的10bit來選擇PTE中相對應的page directory entry，再使用虛擬位址中最後的12bit來取代實體位址的最後12bit。如下圖所示。

struct run {
  struct run *next;
};


//main呼叫kinit1()及kinit2()來初始化allocator
//kinit1()以及kinit2()皆呼叫freerange，且透過page呼叫kfree來增加memory到free list內。
void
kinit1(void *vstart, void *vend)
{
  initlock(&kmem.lock, "kmem");
  kmem.use_lock = 0;
  freerange(vstart, vend);
}

void
kinit2(void *vstart, void *vend)
{
  freerange(vstart, vend);
  kmem.use_lock = 1;
}

void
freerange(void *vstart, void *vend)
{
  char *p;
  p = (char*)PGROUNDUP((uint)vstart);
  for(; p + PGSIZE <= (char*)vend; p += PGSIZE)
    kfree(p);
}

//Free掉page的被v鎖指到實體位址
void
kfree(char *v)
{
  struct run *r;

  if((uint)v % PGSIZE || v < end || V2P(v) >= PHYSTOP)
    panic("kfree");


  memset(v, 1, PGSIZE);

  if(kmem.use_lock)
    acquire(&kmem.lock);
  r = (struct run*)v;
  r->next = kmem.freelist;
  kmem.freelist = r;
  if(kmem.use_lock)
    release(&kmem.lock);
}


//回傳一個指標Kernel可以使用的，若無法allocate則回傳0.
char*
kalloc(void)
{
  struct run *r;

  if(kmem.use_lock)
    acquire(&kmem.lock);
  r = kmem.freelist;
  if(r)
    kmem.freelist = r->next;
  if(kmem.use_lock)
    release(&kmem.lock);
  return (char*)r;
}

int growproc(int n)
{
    uint sz;
