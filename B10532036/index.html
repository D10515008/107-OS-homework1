項目一

xv6有八個32位元通用暫存器%eax,%ebx,%ecx,%edx,%esi,%edi,%ebp,%esp和一個程式計數器%eip(instruction pointer)。
%cs,%ss,%ds,%es,%fs,%gs則是分節暫存器，保存了Segment Selector，讓他能快速被取出。%cs指向含有程式指令的一個節區，%ss指向含有當前程式Stack的一個節區，%ds指向含有全域或靜態資料的暫存器。

項目二
第零章
作業系統的工作有

  1. 多個程式可以分享電腦的資源，並且提供比硬體更有用的服務
  2. 管理和抽象底層硬體
  3. 讓多個程式能像是同時進行
  4. 給程式控制方法讓他們可以分享檔案或是共同工作

Xv6使用傳統概念kernel，一個像執行中程式提供服務的程式。每個執行中的程式稱為process，都有一個包含指令、數據、stack的記憶體空間。Process透過system call使用kernel服務，系統進入kernel，在kernel提供服務後返回，因此process會在用戶空間跟kernel空間切換執行。
一個xv6 process由用戶空間(指令、數據、stack)和僅供kernel可視的process state。Xv6在可以使用的CPU切換，決定哪個等待的process可以執行。當process不在執行，xv6會存他的CPU 暫存器，在他再次執行的時候回復暫存器的值。Kernel 將process跟pid(process identifier)做聯結。

Processes and memory
Process可以通過fork建立一個 child process ，對於parent ，fork 返回child的pid，對於child，fork返回0。可以透過wait等待child 返回，在child使用exit退出。

I/O and File descriptors
File descriptors是一個整數，代表一個可以被process讀寫的文件，可以透過打開文件、目錄、設備、建立一個pipe或複製已經存在的file descriptor來取得。

Pipes
Pipe是一個小kernel buffer，提供給process一對file descriptor，一個用來寫入，一個用來讀取。

File system
Xv6 文件系統提供文件和目錄，文件是一個byte array，目錄包含指向的文件和其他目錄。

第一章
操作系統的核心需求在於同時支持多個活動，因此操作系統必須在這些process中分時共享資源。操作系統必須保證所有進程都正常運行，因此就必須配置資源和隔離進程。也就是說，操作系統必須滿足三個要求：多工，隔離和交互。
為了實現更健壯的隔離，必須能有效禁止應用程序直接訪問電腦資源，以及將資源抽象為服務。
x86的處理器有兩種執行指令的模式，分別是kernel模式和用戶模式。在kernel模式下，處理器允許執行特權指令，比如，硬碟（或者其他I / O設備）讀寫等特權指令。如果程式在用戶模式下嘗試執行特權指令，處理器將拒絕執行指令，必須切換到kernel模式，以便kernel中的軟件將清理該應用程序，因為它做了不該做的事情。運行在用戶模式下的程式只能執行用戶模式指令（比如加減法等），而運行在kernel模式下的軟件能夠執行特權指令。

Code: the first address space

Kernel被load到記憶體後實際上的地址是低地址，所以需要使用page table 來映射到高地址。

Code: creating the first process

main.c在初始化後，呼叫userinit()。 userinit()呼叫allocproc建立第一個process。Allocproc做了一連串設定，分配tf和context空間，並把context中的eip暫存器的值設為forkret()的地址。Userinit()呼叫setupkvm()為process建立page table對映到kernel空間。然後透過inituvm()將process的table地址對映到虛擬地址的0。Userinit將初始process的trapframe裡的%cs、%ds、%es、%ss設為DPL_USER(user mode)，%esp為最大合法虛擬地址，p->sz指到initcode的進入點，地址0。
之後Scheduler找到唯一一個RUNNABLE的process initproc，切換成目標的page table並開始執行。Scheduler叫swtch將%esp設成存在暫存器的p->contest，而這個之前被allocproc設成forkret()的地址，因此開始在forkret()執行。等到forkret()返回，換成trapret執行。Trapret將暫存器的內容還原成trapframe的內容。此時%eip是虛擬地址0，透過page轉成物理地址，第一個process建立完成。

項目三

有三種情形控制權必須從user mode轉換回kernel mode
1.system call:當使用者需要OS的服務
2.exception:當使用者行為不合法時，例如:除以0、存取到不存在的記憶體
3.interrupt:當device發出信號想引起OS的注意，例如:讓kernel做time sharing

如果一個 operating system 有一個處理 interrupt 的 plan，那麼它也可以處理 system call 和 exception，基本動作如下:

  1. 暫停processor loop並開始執行interrupt handler
  2. 儲存被暫停的processor的register(以便障礙排除後回到user mode繼續執行)

x86 protection
x86 有 4 個 protection levels，從 0 (most privilege) ~ 3 (least privilege)，大多數的 operation system 只會使用到 0 (kernel mode) 和 3 (user mode)，而 x86 executes instructions 的 current privilege level 會暫存於 register %cs。
在 x86，interrupt handler 被定義在 interrupt descriptor table(IDT)，IDT 有 256 個 entry，每個 entry 都有 %cs 和 %eip，用來處理相對應的 interrupt。
為了在 x86產生一個 system call，一個 program 呼叫 int n instruction，其中，n 代表 IDT 的 index，intinstruction 實行如下：
• 從 IDT 取得第 n 個 descriptor，其中，n 是 int 的 argumant
• 確認 CPL <= DPL (Descriptor Privilege Level)
• 只在 target segment selector 的 PL < CPL 時，於 CPU 內部 register 存入 %esp 和 %ss
• 從某個 task segment descriptor 載入 %ss 和 %esp
• push %ss
• push %esp
• push %eflags
• push %cs
• push %eip
• 只在 interrupt 的情形下，清除 %eflags 裡的 IF (Instruction Fetch)
• 設定 %cs 和 %eip 到 descriptor 中的 value

Code: Assembly trap handlers
tvinit建立一有256個entry的IDT

tvinit(void)
{
int i; // interrupt(對應到vector[i]找到handler

for(i = 0; i < 256; i++) // IDT有256個entry
SETGATE(idt[i], 0, SEG_KCODE<<3, vectors[i], 0);
SETGATE(idt[T_SYSCALL], 1, SEG_KCODE<<3, vectors[T_SYSCALL], DPL_USER); // Tvinit處理T_SYSCALL，而第二個參數是1，指出閘門是"trap"型別
initlock(&tickslock, "time");
}

tvinit 處理 TSYSCALL，user system 稱它為 trap，特別定義 gate 是一個 trap 的 type，trap gate 不會清除 IF flag，且它在 system 呼叫 handler 時，允許其它的 interrupt
kernel 也有設定一個 system call gate privilege 給 DPL_USER，DPL_USER 允許 user program 產生一個 int instruction 的 trap，然而，xv6 則否，若 xv6 常是如此，會有 general protection excrption 出現，即走向 vector 13

Code: C trap handler

void trap(struct trapframe *tf)
{
    if(tf−>trapno == T_SYSCALL)
    {
        if(myproc()−>killed)
            exit();
        myproc()−>tf = tf;
        syscall();
        if(myproc()−>killed)
            exit();
        return;
    }
    switch(tf−>trapno)
    {
        case T_IRQ0 + IRQ_TIMER:
         if(cpuid() == 0)
         {
            acquire(&tickslock);
            ticks++;
            wakeup(&ticks);
            release(&tickslock);
         }
         lapiceoi();
         break;
        case T_IRQ0 + IRQ_IDE:
         ideintr();
         lapiceoi();
         break;
        case T_IRQ0 + IRQ_IDE+1:
         break;
        case T_IRQ0 + IRQ_KBD:
         kbdintr();
         lapiceoi();
         break;
        case T_IRQ0 + IRQ_COM1:
         uartintr();
         lapiceoi();
         break;
        case T_IRQ0 + 7:
        case T_IRQ0 + IRQ_SPURIOUS:
         cprintf("cpu%d: spurious interrupt at %x:%x\n",
         cpuid(), tf−>cs, tf−>eip);
         lapiceoi();
         break;
        default:
         if(myproc() == 0 || (tf−>cs&3) == 0)
         {
            cprintf("unexpected trap %d from cpu %deip %x (cr2=0x%x)\n",
            tf−>trapno, cpuid(), tf−>eip, rcr2());
            panic("trap");
        }
        cprintf("pid %d %s: trap %d err %d on cpu %d ""eip 0x%x addr 0x%x−−kill proc\n",
        myproc()−>pid, myproc()−>name, tf−>trapno,
        tf−>err, cpuid(), tf−>eip, rcr2());
        myproc()−>killed = 1;
    }
    if(myproc() && myproc()−>killed && (tf−>cs&3) == DPL_USER)
        exit();
    if(myproc() && myproc()−>state == RUNNING &&
       tf−>trapno == T_IRQ0+IRQ_TIMER)
        yield();
    if(myproc() && myproc()−>killed && (tf−>cs&3) == DPL_USER)
        exit();
}

tf−>trapno決定是什麼狀況
在確認 system call 後，trap 會尋找 hardware interrupt
如果 trap 不是一個 system call 也不是interrupt，trap function 會假定是 exception
Code: System calls
syscall從trap frame載入包含%eax的system call number，由%eax索引至對應的system call tables，對第一個system call，%eax包含SYS_exec的值，syscall將會找到system call table上第SYS_exec個索引

void syscall(void)
{
    int num;
    struct proc *curproc = myproc();
    num = curproc−>tf−>eax;
    if(num > 0 && num < NELEM(syscalls) && syscalls[num]) 
    {
        curproc−>tf−>eax = syscalls[num]();
    } 
    else 
    {
        cprintf("%d %s: unknown sys call %d\n",
        curproc−>pid, curproc−>name, num);
        curproc−>tf−>eax = −1;
    }
}

Drivers
driver 是在 operating system 裡，用來處理特定 device 的code，它告訴 device hardware 如何 perform operation、配置 device 產生 interrupt、處理 resulting interrupt。
disk driver 提供一個很好的例子，disk driver 從 disk 複製資料，再回到 disk，而 disk hardware 一般來說會將 disk 上的 data 看作 512 bytes 的 block 數列，其中，block 的 size 通常是 sector size的好幾倍，為了呈現xv6的block，有了struct buf。
