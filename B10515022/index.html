項目二- Chapter 0 and Chapter 1

運作原理及介紹

OS工作簡介:

  • 讓計算機資源在多個程序間共享
  • 管理並虛擬化底層硬體，讓程式不須知道現在使用何種硬體
  • 可以透過輪流使用硬體效能來實現多工
  • 讓應用程式間能夠交流資源互相呼叫或溝通

XV6介紹:

xv6是一款讓介面盡量簡單化，依賴於少量mechanism組出強大功能的基本作業系統，他提供unix的基本介面並模仿unix設計，認識xv6就可以認識最基本的系統，他可以運行在Intel80386(x86)以後的處理器。

運作原理:


每一個運行中的程式都擁有指令、資料、堆疊，指令可以實現運算，資料為運算中的變量，堆疊管理程式的調用，當一個process(行程)要進去kernel服務時需要做system call，等到kernel執行完服務才返回，會在user space跟kernel space交互運行，且Kernel具有cpu保護機制確保用戶只能訪問自己的內存空間，在下面有system call的解說。

System call解說:

名詞解說:

Process(行程)&memory

Process由兩部分(用戶的inst、data、stack)及kernel可見的process，為了達到分時多工在cpu間切換，xv6會賦予每個process一個pid(process identifier)來儲存register值，當要恢復執行時會把他們的register直寫回去還原。

I/O and File descriptors

File descriptor是一個數字，代表一個行程是被寫入或被讀取，只要在打開文件、打開目錄、打開設備、創建pipe時都會得到一個file descriptor。

Pipes

Pipe是kernel的buffer，他可以提供行程兩組file descriptor，一組從pipe一端寫入，另一組從另一端讀取，讓行程能夠溝通聯繫。

File system

Xv6提供資料文件，資料文件內是由幾個byte長度的array還有目錄所組成，而目錄是由樹狀結構所構成，會有其所指向的其他目錄或是自己所引用的目錄，根目錄。

Process


Process 是抽象概念，process會假設自己獨佔一台機器，其他process都無法讀寫的address space還有一顆看起來僅執行本身process的cpu，Xv6使用page table來提供每個process一個自己的記憶體空間，page table會轉換虛擬address至實體address，每個process的address0位址是放user的起始位址，其中指令在前面，接著是全域變數在來是stack，最後是heap區域用來malloc一段記憶體用
Kernal的指令跟data也會被process refer到每個user的memory中，為了留下足夠的memory給user，xv6將process起始位址 refer至memory的高位址0x80100000，xv6使用struct proc來維護一個process，每個process都有一個thread來執行process的指令，其中包含user thread 及 kernel thread，precess運行user指令時只有user thread被使用，kernel thread為空，當進入kernal指令時，user thread仍保存資料，只是暫時不動，prcess的thread就來回在這兩種thread交互移動著。

基本流程

Code: creating the first process


Main初始化一些設備系統後，會創建userinit建立第一個process，userinit會先call allocproc(分配新slot在process table中)，在創建第一個process時會call user init及allocproc，之後每次創建process就都只會call allocproc了，allocproc會在proc表中找到UNUSED的slot，並把他設為EMBRYO，給這個process一個獨有的pid。

Code: Running the first process

在第一個process被設置好後，mpmain會call scheduler開始執行process，scheduler會找到p->state為RUNNABLE的process做initproc，然後將per-cpu變數proc設定為該process，接著調用switchuvm，設置好任務狀態SEG_TSS，接著scheduler把p->state設為RUNNING，調用swth，swtch會保存目前register，把proc->context載入到x86的硬體register，Allocproc透過把initproc的p->context->eip設定為forkret使得ret開始執行forkret，第一次執行時forkret會call一些初始化的函數之後結束，Allocproc接著會執行trapret，此時%esp保存著p->tf，popal恢復通用register，pop1恢復%gs、%fs、%es、%ds，addl跳過trapno和errcode，最後iret跳出%cs、%eip、%flags、%esp、%ss，之後就轉移到%eip的值繼續執行。

The first system call: exec

Initcode.S一開始會call exec，exec會用一個新的process來代替當前process的memory跟register，之後initcode.S會將argv、argv、init、$0推入stack，把%eax設為SYS_exec然後執行intT_SYSCALL，告訴kernel運行exec。

項目三- Chapter 4 Lock

運作原理


xv6 執行於多處理器上。且處理器們共用實體記憶體並共用資料結構，因此可能會造成一顆處理器要讀資料時，另一顆正在寫入資料的問題發生。因此，xv6建立了Lock機制來防止處理器之間相互干擾。
Lock保證了同一時間只有一顆CPU能持有Lock，且持有Lock的CPU才能存取資料，確保資料結構。



















//page52
struct list {
    int data;
    struct list *next;
};

struct list *list = 0;

void
insert(int data)
{
    struct list *l;

    l = malloc(sizeof *l);
    l->data = data;
    l->next = list;
    list = l;
}

在上面的程式碼中若兩個cpu都執行insert函式，則會形成兩個list next node，第16行處cpu1會把cpu2回傳的list結果覆蓋掉，在這稱為race condition。
















//page53
struct list *list = 0; 
struct lock listlock;

void
insert(int data)
{
    struct list *l;
    acquire(&listlock);
    l = malloc(sizeof *l);
    l->data = data;
    l->next = list;
    list = l;
    release(&listlock);
}

解決方法就是使用lock，實作方法如上程式碼

struct spinlock {
uint locked; // 是否lock 0表沒lock

代碼解釋














//spinlock.c
void acquire(struct spinlock *lk)
{
    pushcli(); // 避免被deadlock鎖住
    if(holding(lk)) //CPU是否有lock
        panic("acquire");

    while(xchg(&lk−>locked, 1) != 0);//如果沒有CPU佔用lock，有的話等待lock釋出

    __sync_synchronize();//避免程序被重新排序
    lk−>cpu = mycpu();//紀錄現在佔有lock的CPU
    getcallerpcs(&lk, lk−>pcs);//紀錄現在的call stack
}














//spinlock.c
void release(struct spinlock *lk)
{
    if(!holding(lk))//CPU是否已release
        panic("release");

    lk−>pcs[0] = 0;//釋出lock
    lk−>cpu = 0;//初始化

    __sync_synchronize();//避免被重新排序
    asm volatile("movl $0, %0" : "+m" (lk−>locked) : );
    popcli();//關閉pushcli
}

























//spinlock.c

//acquire時執行pushcli，避免被中斷鎖住
{
void pushcli(void)
    int eflags;

    eflags = readeflags(); //取得flags
    cli();
    if(mycpu()−>ncli == 0) 
        mycpu()−>intena = eflags & FL_IF; 
    mycpu()−>ncli += 1;
}

//release時執行popcli，解除不行中斷的指令
void popcli(void)
{
    if(readeflags()&FL_IF)
        panic("popcli − interruptible");
    if(−−mycpu()−>ncli < 0)
        panic("popcli");
    if(mycpu()−>ncli == 0 && mycpu()−>intena)
        sti();
}
